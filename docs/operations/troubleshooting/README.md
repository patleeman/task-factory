# Troubleshooting

Store issue diagnosis and remediation guides in this directory.

## Naming convention

`symptom-or-alert-name.md` (example: `high-api-latency.md`)

## Authoring steps

1. Copy [`Troubleshooting template`](../../templates/TROUBLESHOOTING_TEMPLATE.md)
2. Document symptoms, signals, root causes, and fixes
3. Add follow-up prevention actions
